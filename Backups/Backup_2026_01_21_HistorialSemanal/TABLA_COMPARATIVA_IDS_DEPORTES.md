## 🔍 Observaciones Clave

### 1. **Inconsistencias de Mapeo**
- **Fitbit** usa IDs muy específicos (ej: 90009 para Running, 15000 para Yoga)
- **Google Fit** usa rango 0-122 secuencial
- **Samsung** usa rangos por categoría (1000s = caminar/correr, 2000s = deportes de bate, etc.)
- **Apple** usa strings descriptivos (más legibles pero más largos)
- **Garmin** usa strings en mayúsculas con guiones bajos

### 2. **Deportes Problemáticos**
Estos deportes tienen mapeos muy diferentes entre plataformas:
- **Gym/Strength Training**: Fitbit (2050), Apple (traditionalStrengthTraining), Samsung (9013)
- **Yoga**: Fitbit (15000), Apple (yoga), Samsung (9010)
- **Soccer**: Google Fit (29), Fitbit (90013), Apple (soccer), Samsung (4004)

### 3. **ID Genérico "Other"**
- **Google Fit**: 108
- **Samsung**: 0
- **Huawei**: 255
- **Apple**: "other"

### 4. **Plataformas Sin API Pública**
- **Xiaomi**: Solo 2 IDs documentados (1=Walking, 7=Normal steps)
- **Huawei**: Documentación limitada, solo 6 IDs básicos

## 💡 Recomendaciones para Wellnessfy

### Estrategia de Mapeo:
1. **Crear diccionario maestro** con nombres estándar de Wellnessfy
2. **Mapear cada plataforma** a nombres estándar
3. **Usar nombre de la plataforma** cuando no haya mapeo directo

### Ejemplo de Implementación:
```javascript
const WELLNESSFY_SPORTS = {
  'yoga': {
    googleFit: 100,
    fitbit: 15000,
    apple: 'yoga',
    samsung: 9010,
    garmin: 'YOGA',
    huawei: null,
    xiaomi: null
  },
  'gym': {
    googleFit: null,
    fitbit: 2050,
    apple: 'traditionalStrengthTraining',
    samsung: 9013,
    garmin: 'STRENGTH_TRAINING',
    huawei: null,
    xiaomi: null
  }
  // ...
};
```

### Fallback Strategy:
- Si el ID no está en el diccionario → Usar nombre original de la plataforma
- Si la plataforma usa ID genérico (108, 0, 255) → Intentar obtener nombre de Sessions API
- Si no hay nombre → Mostrar "Actividad Física" en lugar de "Desconocida"
