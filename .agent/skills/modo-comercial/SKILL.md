# Skill: Modo Comercial 🚀💰

Este Skill transforma la planeación técnica en una **estrategia de producto**. Su objetivo es asegurar que cada línea de código contribuya a la sostenibilidad comercial de Wellnessfy, diferenciando claramente entre el valor para el usuario final (B2C) y el valor para empresas/organizaciones (B2B).

## 🎯 Objetivos Estratégicos
1. **Monetización**: Identificar qué funciones son el "gancho" (Free) y cuáles son el "valor" (Premium).
2. **Segmentación**: Diferenciar la experiencia para el usuario individual vs. el entorno corporativo.
3. **Conversión**: Diseñar flujos que inviten al usuario a escalar de Freemium a Premium.
4. **Valor Percibido**: Asegurar que las funciones premium se sientan realmente exclusivas y superiores.

---

## 🏗️ Protocolo de Planeación Comercial

Cuando se active el **Modo Comercial**, cada propuesta técnica DEBE responder a los siguientes pilares antes de proceder a la implementación:

### 1. Clasificación del Modelo (Pricestack)
*   **🆓 Freemium (Core)**: Funciones básicas necesarias para que la app sea útil y genere el hábito.
    *   *Ejemplo*: Registro de pasos, acceso a desafíos básicos, feed social estándar.
*   **💎 Premium (Suscripción)**: Funciones avanzadas que ahorran tiempo, dan estatus o proporcionan insights profundos.
    *   *Ejemplo*: Sincronización automática ilimitada, analítica avanzada de salud, desafíos exclusivos de marca, personalización de avatar avanzada.

### 2. Vertiente de Mercado (Targeting)
*   **👤 B2C (Consumer)**: Enfoque en gamificación, redes sociales, retos virales y autorrealización.
    *   *KPI*: Retención diaria y compartidos.
*   **🏢 B2B (Business)**: Enfoque en dashboards para HR, retos corporativos, reporting de bienestar grupal y marca blanca.
    *   *KPI*: LTV (Life Time Value) y reducción de rotación laboral.

### 3. El "Trigger" de Venta
*   ¿En qué momento esta función le dice al usuario "Necesitas la versión Pro"?
*   ¿Cómo se presenta el límite de la versión gratuita de forma elegante (sin romper la UX)?

---

## 🛠️ Checklist para Nuevas Funciones

Para cualquier requerimiento, el análisis debe incluir:

1.  **Segmento Relevante**: ¿A quién beneficia más? (B2C / B2B / Ambos).
2.  **Tier de Acceso**: ¿Es una función Free o Premium?
3.  **Valor de Conversión**: Si es Premium, ¿cuál es el argumento de venta? Si es Free, ¿cómo ayuda a que el usuario eventualmente compre?
4.  **Upsell Points**: Puntos en la interfaz donde se mostrará la opción de desbloquear o mejorar esta función.
5.  **Visión Comercial del Feature**: Si no se especifican detalles comerciales, se asume que la función debe diseñarse para ser **comercializable** (por ejemplo, con límites de uso o tags de "Pro").

---

## 💡 Mentalidad: "No solo construimos, vendemos"
*   **Estética Premium**: Las funciones de pago deben verse visualmente más ricas (glasmorfismo más acentuado, animaciones exclusivas).
*   **Escalabilidad B2B**: Pensar siempre si esta función podría ser administrada por un "Líder de Círculo" o un "Gerente de Bienestar" en una empresa.
*   **Cero Placeholders**: En modo comercial, los ejemplos de datos deben evocar abundancia y éxito profesional/físico.
